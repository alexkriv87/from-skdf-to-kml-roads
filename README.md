# from_skdf_to_kml_roads

Скрипт для выгрузки дорог из СКДФ по координатам двух углов прямоугольника.

## Возможности

- Принимает координаты в двух форматах:
  - Яндекс.Карты: `55.972483, 36.911828`
  - SAS.Планет: `N51°43'15.9790" E121°07'42.0984"`
- Автоматически определяет формат
- Запрашивает zoom (уровень детализации)
- Скачивает все дороги в указанном прямоугольнике
- Обогащает дороги характеристиками (категория, покрытие)
- Сохраняет результат в KML с цветовой кодировкой по принадлежности:
  - Федеральные — коричневый
  - Региональные — голубой
  - Местные — фиолетовый

## Как это работает

1. Пользователь вводит две точки (северо-запад и юго-восток)
2. Скрипт строит прямоугольник (bbox)
3. Запрашивает у API СКДФ все дороги в этом прямоугольнике
4. Для каждой дороги получает характеристики (через passport_id)
5. Формирует KML-файл с цветами по принадлежности

## Структура проекта

| Файл | Назначение |
|------|-----------|
| `main.py` | Основной скрипт, диалог с пользователем |
| `coord_utils.py` | Парсинг координат (Яндекс и SAS.Планет) |
| `skdf_api.py` | Запросы к API СКДФ |
| `logger_config.py` | Настройка логирования |
| `kml_exporter.py` | Формирование KML-файла |
